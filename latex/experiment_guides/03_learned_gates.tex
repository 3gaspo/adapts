\section{Theoretical overview and goal}
Retrieval can help some queries and harm others.  A gate chooses between the
vanilla forecast $\hat y_0$ and the context forecast $\hat y_c$.  On \Ttwo the
supervision signal is the loss advantage
\[
 \Delta_h=(y_h-\hat y_{0,h})^2-(y_h-\hat y_{c,h})^2,
\]
so $\Delta_h>0$ means context is preferable.  Scalar gates make one decision
per window; horizon gates make one decision per forecast step.  Regressors
estimate $\Delta$, while classifiers estimate the event $\Delta>0$.

\goal{Learn when retrieval should be trusted, quantify the value of query-level
features, and compare attainable gates with target-aware oracle upper bounds.}

\section{Implementation details}
The gate features summarize the context--vanilla difference, query mean and
standard deviation, neighbor look-back statistics, same-user ratio, neighbor
age, distance-weight concentration, and average distance.  Horizon gates also
receive the full horizon-wise context difference.

\midrule

CatBoost uses 300 iterations, learning rate 0.03, depth 4, balanced classes,
and seed 1.  Models are trained only on \Ttwo and applied without refitting to
\Tthree.  The publication retrieval grid is raw/instance Euclidean distance
with $k\in\{1,3,10\}$ across the seven datasets, eight settings, and Chronos plus
TabPFN-TS defined by the extraction sweep.  The 572--64 setting is retained for
the Cross-RAG comparison; TS-ICL remains a later extension.

Outputs
include per-method metrics, prediction payloads, gate diagnostics, and feature

The gate job is an array and validates each extraction completion manifest
before loading it.  It must depend on the extraction array.  The table job then
checks every requested gate JSON before producing detailed and averaged tables
for each backbone; missing configurations are errors rather than silent blanks.

\section{Interpretation}
The oracle gap measures headroom, not deployable performance.  The comparison
between the no-feature Bayes gate and feature-based gates isolates the benefit
of conditional decisions.  Final claims must use \Tthree and should report both
mean error and user-tail error because a gate can improve the mean while
systematically rejecting useful context for a subgroup.
